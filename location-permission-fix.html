iOS Location Permissions Error

Fixing the NSLocation*UsageDescription keys missing in your React Native app

Error Message

Error requesting location permissions: Error: One of the `NSLocation*UsageDescription` keys must be present in info.plist to be able to use geolocation.

This error occurs when your React Native app tries to access location services without the required privacy descriptions in your iOS project's Info.plist file.

Step-by-Step Solution

1
Open your iOS project in Xcode

Navigate to your React Native project's iOS directory and open the .xcworkspace file in Xcode.

2
Select your project in the navigator

Click on your project name in the left sidebar, then select your target and go to the "Info" tab.

3
Add the required keys to Info.plist

Add the following keys to your Information Property List:

<key>NSLocationWhenInUseUsageDescription</key> <string>We need your location to find parking spots near you.</string> <key>NSLocationAlwaysAndWhenInUseUsageDescription</key> <string>We need your location to find parking spots even when the app is in the background.</string>
4
Add background mode capability

In Xcode, select your target, go to "Signing & Capabilities", click "+ Capability" and add "Background Modes". Check "Location updates" if needed.

5
Rebuild your project

Clean and rebuild your iOS project to apply the changes.

cd ios xcodebuild clean cd .. npx react-native run-ios

Additional Information

Location Usage Types

  • NSLocationWhenInUseUsageDescription - App needs location only when in use
  • NSLocationAlwaysAndWhenInUseUsageDescription - App needs location even when in background
  • NSLocationAlwaysUsageDescription - For apps supporting iOS 10 and earlier (deprecated)
Important: Apple requires clear and honest explanations for why your app needs location data. Apps may be rejected if the purpose isn't clear to users.

Testing Location Permissions

After making these changes, test your app on a real iOS device (location permissions don't work reliably on simulators).

Pro Tip: Use the react-native-permissions library to handle location permissions in a more streamlined way across both iOS and Android.

Example LocationService.js Fix

Here's how to properly handle location permissions in your JavaScript code:

import { PermissionsAndroid, Platform } from 'react-native'; import Geolocation from 'react-native-geolocation-service'; async function requestLocationPermission() { if (Platform.OS === 'ios') { const auth = await Geolocation.requestAuthorization('whenInUse'); return auth === 'granted'; } if (Platform.OS === 'android') { const granted = await PermissionsAndroid.request( PermissionsAndroid.PERMISSIONS.ACCESS_FINE_LOCATION ); return granted === PermissionsAndroid.RESULTS.GRANTED; } } View React Native Geolocation Documentation
